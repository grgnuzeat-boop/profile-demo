Full Stack Development

Front-End · Back-End · Database — understanding the complete web development picture and how all layers connect.

HTML5CSS3JavaScript ReactNode.jsExpress MongoDBGit
What is Full Stack Development?

Full Stack Development means building both the client-facing side of a web application (the front-end) and the server-side logic and data management (the back-end). A Full Stack Developer understands the complete journey of a web request — from a user clicking a button, to a server processing logic, to a database storing the result.

Common stacks include the MERN stack (MongoDB, Express, React, Node.js) and the LAMP stack (Linux, Apache, MySQL, PHP). The key advantage is the ability to own an entire feature — writing the UI, the API, and the database schema without handoffs.

🖥 Frontend (Client)HTML · CSS · Bootstrap · JavaScript · React
↕ HTTP / REST / WebSocket
⚙️ Backend (Server)Node.js · Express · Python · Django
↕ SQL / NoSQL Queries
🗄 DatabaseMongoDB · PostgreSQL · MySQL · Redis
↕ CI/CD · Containerisation
☁️ DevOps / HostingDocker · Vercel · Render · GitHub Actions · AWS

HTML5 — HyperText Markup Language

The skeleton of every web page. HTML defines the semantic structure and meaning of content through a system of nested tags.

What is HTML?

HTML (HyperText Markup Language) is the foundational language of the web. It uses elements and tags to define the structure and meaning of web content. Every website is built on HTML at its core.

HTML5 introduced semantic elements like <article>, <section>, <nav> and <footer> — giving meaning beyond appearance and helping screen readers and search engines understand page structure.

<header><nav><main> <section><article><aside> <footer><figure><time>
Example 1 — Document Structure

Every HTML page follows the same fundamental skeleton. <!DOCTYPE html> tells the browser to use standards mode. The <head> holds metadata; the <body> holds visible content.

index.html

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Tojang Gurung — Portfolio</title>
</head>
<body>
  <h1>Hello, I'm Tojang Gurung</h1>
  <p>Welcome to my web page!</p>
</body>
</html>

CSS3 — Cascading Style Sheets

CSS controls the visual presentation of HTML — layout, colour, typography, spacing, animations and responsive design.

What is CSS?

CSS (Cascading Style Sheets) controls the visual presentation of HTML — colours, fonts, spacing, layout and animations. Without CSS, every website would look like a plain text document.

CSS works by selecting HTML elements and applying declarations (property: value pairs). The "cascading" means multiple rules can apply to one element, with specificity and source order determining which wins.

Example 1 — Typography Styling

Control font size, colour, alignment and style. CSS gives precise control over every typographic detail, from line-height to letter-spacing.

typography.css

  font-size:  2rem;
  font-weight:700;
}
p {

What is Bootstrap?

Bootstrap is a free, open-source CSS framework originally built at Twitter. It gives developers pre-built components — navbars, buttons, cards, alerts, grids — so you can build responsive, mobile-first websites much faster than writing CSS from scratch.

Bootstrap uses a 12-column Flexbox grid and utility classes like d-flex, mt-3, text-center and p-4 for instant layout control directly in your HTML, keeping stylesheets lean.

JavaScript — Dynamic Behaviour

JavaScript adds interactivity and logic to web pages — from DOM manipulation to async API calls, loops, functions and full applications.

VariablesFunctionsLoopsDOM ManipulationEventsAsync / FetchES6+Arrays & Objects
What is JavaScript?

JavaScript (JS) is a lightweight, interpreted programming language originally created to make web pages interactive. It now runs everywhere — in browsers and on servers via Node.js — making it the world's most widely deployed programming language.

JS lets you dynamically update content, animate elements, respond to user events and communicate with back-end APIs — all without reloading the page.

Example 1 — Variables & Output

Use const for fixed values and let for reassignable ones. Template literals (backticks with ${}) embed expressions directly into strings.

variables.js
const name   = "Tojang Gurung";
const role   = "Full Stack Developer";
let   score  = 92;
const active = true;

// Template literal — embeds variables in a string
console.log(`Name: ${name}`);
console.log(`Role: ${role}`);
console.log(`Score: ${score} / 100`);
console.log(`Available: ${active}`);

Example 2 — Loops & Arrays

A for loop repeats code a set number of times. Modern JS array methods like .map(), .filter() and .forEach() provide cleaner, more expressive iteration.

loops.js
const skills = ["HTML", "CSS", "JavaScript", "Node.js"];

// Classic for loop
for (let i = 0; i < skills.length; i++) {
  console.log(`${i + 1}. ${skills[i]}`);
}

// .map() — transform each item into uppercase
const upper = skills.map(s => s.toUpperCase());
console.log(upper);

Example 6 — Todo App (DOM + Array State)

A complete mini-app demonstrating array state management, dynamic DOM creation and event handling. Every action — add, complete, delete — updates a JS array and re-renders the list.

todo-app.js
let todos = [];

function addTodo(text) {
  todos.push({ id: Date.now(), text, done: false });
  render();
}
function toggle(id) {
  const t = todos.find(t => t.id === id);
  if (t) t.done = !t.done;
  render();
}
function remove(id) {
  todos = todos.filter(t => t.id !== id);
  render();
}
function render() {
  list.innerHTML = todos.map(t => `
    <div class="todo-item">...</div>
  `).join('');
}
